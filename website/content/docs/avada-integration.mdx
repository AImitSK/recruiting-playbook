Recruiting Playbook bietet native Fusion Builder Elemente für Avada-Nutzer. Die Elemente sind als **Pro-Feature** verfügbar und integrieren sich nahtlos in den Backend- und Live-Builder.

## Voraussetzungen

- **Avada Theme** (aktiv)
- **Fusion Builder** Plugin (wird mit Avada mitgeliefert)
- **Recruiting Playbook Pro** Lizenz

## Verfügbare Elemente

Alle Elemente befinden sich in der Kategorie **Recruiting Playbook** im Element-Picker.

![Element-Picker mit allen Recruiting Playbook Elementen](/screenshots/avada-01.png)

### Seiten-Elemente

Diese Elemente können auf jeder Seite verwendet werden:

| Element | Shortcode | Beschreibung | Feature |
|---------|-----------|--------------|---------|
| RP: Stellenliste | `rp_jobs` | Stellenanzeigen im Grid-Layout | Pro |
| RP: Stellensuche | `rp_job_search` | Suchformular mit Filtern | Pro |
| RP: Stellen-Zähler | `rp_job_count` | Dynamischer Zähler für Headlines | Pro |
| RP: Featured Jobs | `rp_featured_jobs` | Hervorgehobene Stellen | Pro |
| RP: Neueste Stellen | `rp_latest_jobs` | Aktuellste Stellenanzeigen | Pro |
| RP: Job-Kategorien | `rp_job_categories` | Klickbare Kategorie-Karten oder -Liste | Pro |
| RP: KI-Job-Finder | `rp_ai_job_finder` | Lebenslauf-Upload mit KI-Matching | AI-Addon |

### Layout-Komponenten (Template Builder)

Diese Elemente sind nur im **Template Builder** (Layout Sections) verfügbar und für Stellen-Einzelseiten gedacht:

| Element | Shortcode | Beschreibung | Feature |
|---------|-----------|--------------|---------|
| RP: KI-Job-Match | `rp_ai_job_match` | "Passe ich zu diesem Job?" Button | AI-Addon |

<Callout type="info">
Layout-Komponenten erkennen die aktuelle Stelle automatisch. Sie funktionieren nur in Layout Sections für den Post-Typ "Stellenanzeige".
</Callout>

<Callout type="info">
Das Bewerbungsformular wird automatisch auf Stellen-Einzelseiten eingebunden und steht als [Shortcode](/docs/shortcodes#rp_application_form) zur Verfügung. Ein separates Fusion Builder Element ist nicht nötig.
</Callout>

## Elemente einfügen

### Backend Builder

1. Seite bearbeiten → **Fusion Builder** aktivieren
2. Auf **Element** klicken
3. Kategorie **Recruiting Playbook** wählen
4. Element auswählen und konfigurieren

### Live Builder

1. Seite bearbeiten → **Live Builder** starten
2. Auf das **+** Symbol klicken
3. Nach "RP:" suchen oder Kategorie **Recruiting Playbook** wählen
4. Element per Drag & Drop platzieren
5. Parameter werden sofort in der Live-Vorschau angezeigt

![Live Builder mit RP: Stellensuche und Element Options](/screenshots/avada-02.png)

## RP: Stellenliste

Zeigt Stellenanzeigen in einem konfigurierbaren Grid-Layout.

### Einstellungen

| Einstellung | Beschreibung | Standard |
|-------------|--------------|----------|
| Anzahl Stellen | Wie viele Stellen angezeigt werden (1-50) | `10` |
| Spalten | Grid-Spalten (1-4) | `2` |
| Kategorie | Filter nach Job-Kategorie | Alle |
| Standort | Filter nach Standort | Alle |
| Beschäftigungsart | Filter nach Art | Alle |
| Nur Featured | Nur hervorgehobene Stellen | Nein |
| Sortieren nach | Datum, Titel oder Zufall | Datum |
| Reihenfolge | Aufsteigend oder Absteigend | Absteigend |

## RP: Stellensuche

Suchformular mit Filtern und Ergebnisliste.

### Einstellungen

| Einstellung | Beschreibung | Standard |
|-------------|--------------|----------|
| Suchfeld anzeigen | Freitext-Suche aktivieren | Ja |
| Kategorie-Filter | Dropdown für Kategorien | Ja |
| Standort-Filter | Dropdown für Standorte | Ja |
| Beschäftigungsart-Filter | Dropdown für Arten | Ja |

## RP: Stellen-Zähler

Dynamischer Zähler für Headlines wie "Aktuell 12 offene Stellen".

### Einstellungen

| Einstellung | Beschreibung | Standard |
|-------------|--------------|----------|
| Kategorie | Nur Stellen dieser Kategorie zählen | Alle |
| Standort | Nur Stellen an diesem Standort zählen | Alle |
| Format (Mehrzahl) | Text mit `{count}` Platzhalter | `{count} offene Stellen` |
| Format (Einzahl) | Text für eine Stelle | `{count} offene Stelle` |
| Format (Null) | Text für keine Stellen | `Keine offenen Stellen` |

## Design & Branding

Die Elemente übernehmen automatisch die Einstellungen aus dem **Design & Branding** Bereich des Plugins:

- Farben und Buttons
- Card-Layouts (Standard, Kompakt, Featured)
- Grid-Einstellungen

Zusätzlich werden bei aktivem Avada Theme die **Button-Styles aus den Avada Global Options** automatisch auf die Plugin-Buttons übertragen.

## Layout Section für Stellenanzeigen

Um den KI-Job-Match-Button auf Stellen-Einzelseiten zu platzieren:

1. **Avada** → **Layouts** → Neues Layout erstellen
2. Bedingung: Post-Typ = **Stellenanzeige** (`job_listing`)
3. **Content Layout Section** erstellen
4. Im Builder die Layout-Komponente (KI-Job-Match) platzieren
5. Layout veröffentlichen

<Callout type="info">
Die Layout-Komponenten sind nur im Template Builder sichtbar, nicht beim Bearbeiten normaler Seiten. So wird versehentliches Platzieren auf unpassenden Seiten verhindert.
</Callout>
