Recruiting Playbook bietet native Fusion Builder Elements fuer Avada-Nutzer. Die Elemente sind als **Pro-Feature** verfuegbar und integrieren sich nahtlos in den Backend- und Live-Builder.

## Voraussetzungen

- **Avada Theme** (aktiv)
- **Fusion Builder** Plugin (wird mit Avada mitgeliefert)
- **Recruiting Playbook Pro** Lizenz

## Verfuegbare Elemente

### Seiten-Elemente

Diese Elemente koennen auf jeder Seite verwendet werden:

| Element | Shortcode | Beschreibung | Feature |
|---------|-----------|--------------|---------|
| RP: Stellenliste | `rp_jobs` | Stellenanzeigen im Grid-Layout | Pro |
| RP: Stellensuche | `rp_job_search` | Suchformular mit Filtern | Pro |
| RP: Stellen-Zaehler | `rp_job_count` | Dynamischer Zaehler fuer Headlines | Pro |
| RP: Featured Jobs | `rp_featured_jobs` | Hervorgehobene Stellen | Pro |
| RP: Neueste Stellen | `rp_latest_jobs` | Aktuellste Stellenanzeigen | Pro |
| RP: Job-Kategorien | `rp_job_categories` | Klickbare Kategorie-Karten | Pro |
| RP: KI-Job-Finder | `rp_ai_job_finder` | Lebenslauf-Upload mit KI-Matching | AI-Addon |

### Layout-Komponenten (Template Builder)

Diese Elemente sind nur im **Template Builder** (Layout Sections) verfuegbar und fuer Stellen-Einzelseiten gedacht:

| Element | Shortcode | Beschreibung | Feature |
|---------|-----------|--------------|---------|
| RP: Bewerbungs-Formular | `rp_application_form` | Mehrstufiges Bewerbungsformular | Pro |
| RP: KI-Job-Match | `rp_ai_job_match` | "Passe ich zu diesem Job?" Button | AI-Addon |

<Callout type="info">
Layout-Komponenten erkennen die aktuelle Stelle automatisch. Sie funktionieren nur in Layout Sections fuer den Post-Typ "Stellenanzeige".
</Callout>

## Elemente einfuegen

### Backend Builder

1. Seite bearbeiten → **Fusion Builder** aktivieren
2. Auf **Element** klicken
3. Kategorie **Recruiting Playbook** waehlen
4. Element auswaehlen und konfigurieren

### Live Builder

1. Seite bearbeiten → **Live Builder** starten
2. Auf das **+** Symbol klicken
3. Nach "RP:" suchen oder Kategorie **Recruiting Playbook** waehlen
4. Element per Drag & Drop platzieren
5. Parameter werden sofort in der Live-Vorschau angezeigt

## RP: Stellenliste

Zeigt Stellenanzeigen in einem konfigurierbaren Grid-Layout.

### Einstellungen

| Einstellung | Beschreibung | Standard |
|-------------|--------------|----------|
| Anzahl Stellen | Wie viele Stellen angezeigt werden (1-50) | `10` |
| Spalten | Grid-Spalten (1-4) | `2` |
| Kategorie | Filter nach Job-Kategorie | Alle |
| Standort | Filter nach Standort | Alle |
| Beschaeftigungsart | Filter nach Art | Alle |
| Nur Featured | Nur hervorgehobene Stellen | Nein |
| Sortieren nach | Datum, Titel oder Zufall | Datum |
| Reihenfolge | Aufsteigend oder Absteigend | Absteigend |

| Einstellung | Beschreibung | Standard |

| Einstellung | Beschreibung | Standard |
|-------------|--------------|----------|

| Format (Null) | Text fuer keine Stellen | `Keine offenen Stellen` |

## Design & Branding

Die Elemente uebernehmen automatisch die Einstellungen aus dem **Design & Branding** Bereich des Plugins:

- Farben und Buttons
- Card-Layouts (Standard, Kompakt, Featured)
- Grid-Einstellungen

Zusaetzlich werden bei aktivem Avada Theme die **Button-Styles aus den Avada Global Options** automatisch auf die Plugin-Buttons uebertragen.

## Layout Section fuer Stellenanzeigen

Um das Bewerbungsformular und den KI-Job-Match-Button auf Stellen-Einzelseiten zu platzieren:

1. **Avada** → **Layouts** → Neues Layout erstellen
2. Bedingung: Post-Typ = **Stellenanzeige** (`job_listing`)
3. **Content Layout Section** erstellen
4. Im Builder die Layout-Komponenten (Bewerbungs-Formular, KI-Job-Match) platzieren
5. Layout veroeffentlichen

<Callout type="info">
Die Layout-Komponenten sind nur im Template Builder sichtbar, nicht beim Bearbeiten normaler Seiten. So wird versehentliches Platzieren auf unpassenden Seiten verhindert.
</Callout>
